# Plum Health Insurance Claims Processing System

Multi-agent AI pipeline that automates health insurance claim submission, document verification, extraction, policy evaluation, and explainable decision-making.

## Free Deployment (Render)

You can deploy this repo for free on Render (free services sleep after inactivity and cold-start on next request). See Render docs: [`https://render.com/docs/free.md`](https://render.com/docs/free.md).

1. Go to Render dashboard and choose **New → Blueprint**
2. Select this GitHub repo: `https://github.com/pveerrotwal/Plum_Assignment`
3. Render will detect `render.yaml` and create a **free** Web Service
4. After deploy completes, your live link will be shown in the Render service page

## Quick Start

### Prerequisites
- Python 3.11+
- Node.js 18+ (for frontend)

### Backend

```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Open http://localhost:5173 — the Vite dev server proxies `/api` to the backend.

### Run Tests & Eval Report

```bash
cd backend
source venv/bin/activate
pytest -v
python scripts/run_eval.py
```

The eval report is written to `EVAL_REPORT.md` at the project root.

### Optional: OpenAI Vision Extraction

Set `OPENAI_API_KEY` to enable real document extraction from uploaded images/PDFs. Without it, the system uses filename heuristics and metadata (sufficient for the 12 eval test cases which provide structured content).

## API Endpoints

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/health` | Health check |
| GET | `/api/policy` | Full policy configuration |
| GET | `/api/members` | Member roster |
| GET | `/api/document-requirements/{category}` | Required docs per claim type |
| POST | `/api/claims/submit` | Submit claim (JSON body) |
| POST | `/api/claims/submit-with-files` | Submit claim with file uploads |
| GET | `/api/claims` | List processed claims |
| GET | `/api/claims/{id}` | Get claim decision + full trace |

## Pipeline Agents

1. **DocumentGatekeeperAgent** — Early validation: required docs, quality, patient consistency
2. **DocumentExtractorAgent** — Structured field extraction from documents
3. **FraudDetectionAgent** — Same-day / high-value pattern detection
4. **PolicyEvaluatorAgent** — Waiting periods, exclusions, pre-auth, financial calculations
5. **DecisionAgent** — Final decision assembly with confidence scoring
6. **PipelineOrchestrator** — Coordinates agents, handles failures gracefully

## Demo Scenarios

- **Early block (TC001):** Submit two prescriptions for a CONSULTATION claim → specific error naming missing Hospital Bill
- **Full approval (TC004):** Valid consultation with prescription + bill → APPROVED ₹1,350 with co-pay trace
- **Component failure (TC011):** Set `simulate_component_failure: true` → APPROVED with degraded confidence and manual review flag
